## OCR Implementation Details
User approved installing Tesseract (system binary) and proceeding. Steps taken:
1. Installed Tesseract OCR 5.5.0 (official `tesseract-ocr/tesseract` GitHub release installer, silent `/VERYSILENT`) — landed at `D:\Programs\Tesseract-OCR\` (this machine's existing Inno Setup default dir, not `C:\Program Files`).
2. Downloaded `tha.traineddata` (official `tesseract-ocr/tessdata` repo) into its `tessdata/` folder — the installer only ships `eng` + `osd` by default, and the source documents are virtually all Thai.
3. `pip install pytesseract` (Pillow was already present).
4. Added `extract_pptx_images_ocr()` to `rag_builder.py`: opens the pptx as a zip, OCRs every image under `ppt/media/` with `lang='tha+eng'`, hardcodes `pytesseract.pytesseract.tesseract_cmd` to the known install path (more robust than relying on a PATH env-var refresh).
5. Wired it as a fallback in the main build loop: only runs when the normal `<a:t>` text-layer extraction yields ≤50 chars.
6. **Found and fixed a real OCR quality issue**: Tesseract's Thai model segmented the dense screenshot text character-by-character (e.g. `ก ล ุ ่ ม ร า ย ง า น` instead of `กลุ่มรายงาน`), which would have made the output useless for the app's substring-based RAG search. Added a regex post-process collapsing spaces between adjacent Thai characters (Thai script has no inter-letter spacing) — confirmed it turns the output into real, legible, matchable Thai phrases (e.g. "กลุ่มรายงานมาตรฐาน", "การเข้าถึงบริการ", "สาขาโรคไม่ติดต่อ (NCD DM,HT,CVD)").
7. Re-ran `rag_builder.py`: knowledge base grew from 1,812 → 1,818 chunks (6 new chunks from the OCR'd PPTX; some duplicate-text chunks were deduped as usual).
8. Deleted the downloaded installer `.exe` after use; did not commit it.
